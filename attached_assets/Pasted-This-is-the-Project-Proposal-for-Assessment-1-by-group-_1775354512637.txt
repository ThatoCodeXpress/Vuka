NAME OF SYSTEM, INTRODUCTION
Smart Dual-Verification Alarm System
A React Native mobile application that forces users to physically verify they are awake by capturing a photo of a
specified object and, thirty minutes later, a morning activity. The system uses TensorFlow.js and Google
Teachable Machine for real-time AI photo verification, making it impossible to dismiss the alarm without
genuinely waking up and completing a morning action.
OBJECTIVE / VISION IDE: Visual Studio Code
Project Objective
This project aims to develop a Smart Dual-Verification Alarm System that solves the universal problem
of oversleeping and alarm snoozing. Traditional alarms are trivially dismissed with a single tap, causing
users to fall back asleep and miss important commitments. This system enforces wakefulness through
two mandatory verification stages:
Stage 1 (Primary Alarm): The user must physically get up, locate a randomly selected object in their
home, and photograph it before the alarm can be silenced.
Stage 2 (Secondary Alarm – 30 minutes later): The user must photograph proof of a morning activity
(e.g., brushing teeth, eating breakfast, or making the bed) to confirm they have transitioned into their
daily routine.
The system integrates TensorFlow.js AI to automatically validate the photos, making photospoofing
impossible. An additional Sleep Score feature tracks user compliance and assigns a daily productivity
score based on wake-up performance. Potential users include students, working professionals, athletes,
and anyone requiring discipline in their morning routine.

ADDITIONAL INNOVATIVE FEATURES [BONUS VALUE-ADDS]
The following features distinguish this system from standard alarm apps and competing student projects,
ensuring uniqueness and technical depth:
► Sleep Score Engine: A proprietary scoring algorithm (0–100) calculates a daily Sleep Score based on: time
between alarm set and actual wake time, number of failed photo verification attempts, time gap between
primary and secondary alarm completion, and how early/late relative to the set target. Scores are stored in
SQLite and displayed as a 7-day trend graph.
► Randomised Anti-Cheat Object Selection: Objects for photo verification are randomly selected each morning
from a personal library the user builds. The randomisation prevents users from pre-positioning objects near their
bed. The app can also require the object to be photographed in a specific room (detected via GPS or room tag).
► Strict Mode (No Snooze Lock): A voluntary "Strict Mode" completely disables snooze. Once Strict Mode is
active, the alarm cannot be dismissed without completing both verification stages. This mode is designed for
users who need extreme accountability.
► TensorFlow.js AI Confidence Scoring: Using a model trained via Google Teachable Machine, the AI engine
returns a confidence score for each submitted photo. If confidence is below 75%, the photo is rejected and the
alarm continues. Users can retrain the model on their own objects for personalised accuracy.
► Accountability Partner Alerts: Users can nominate a trusted contact (e.g., parent, friend, study partner). If
the primary alarm is not dismissed within 15 minutes, the system automatically sends an SMS/WhatsApp
notification to the accountability partner.
► Morning Routine Streak Tracker: The system tracks consecutive days of successful dual verification and
rewards streaks with in-app achievement badges. This gamification element reinforces habit formation and
provides long-term motivation for maintaining discipline.
► Fully Offline Operation: Unlike cloud-dependent competitors, the entire verification pipeline (AI model,
SQLite database, camera, alarm engine) runs entirely offline on the device. This means the app works on
aeroplanes, in areas with no signal, or with mobile data turned off — no privacy concerns about photo uploads.
